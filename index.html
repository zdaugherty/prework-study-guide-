HTML

  • HTML is the universal markup language for the Web.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • h1: Tag represents the heading.
  • This tag defines the document type and HTML version.
  • head:This tag represents the document's header which can keep other HTML tags like title,link etc.
  • title:the title tag is used indside the head tag to mention the document title
  • Body:This tag represents the document's body which keeps other HTML tags like h1,div, p etc.
  • class: HTML elements can have one or more classes, separated by spaces. You can style elements using CSS by selecting them with their classes.
  • id:An HTML element can have an id attribute to identify it. id elements should always be unique to that single element, and each element should never have more than one id.
  • href:Links tell the browser where to go using an href attribute, which stores a URL.
  • Basic Formatting:You can easily format text to be bold, italic, or underlined using simple formatting tags.
  • Comments:HTML comments are sometimes used in code to explain parts of the markup. They are similar to comments in other languages. Users do not see comments in their browser.
  • Div:A block level container (or ‘division’ of the web page) for content with no semantic meaning.
  • Horizontal rules:This tag creates a black line one pixel thick that runs the all the way across its container. It can be styled to look differently with CSS.
  • Lists:HTML supports two kinds of lists: ordered lists and unordered lists. Within lists each individual list item has its own tag.
  • Unordered Lists:Unordered lists are just lists whose items are denoted with bullet points.
  • Ordered Lists:Ordered lists’ items are denoted with numbers.
  • Tables:An element for displaying information in rows and columns. Supports headers and footers for labeling columns. Divides information into rows (denoted by the tr tag) which contain cells (denoted by the td tag).

CSS

  • CSS stands for Cascading style sheets
  • Style sheets come in three types.
  • Incline: style attributes are used inline with the HTML tags.
  • Embedded (internal)-style information is in a style tag which is nested in the head tag of a single web page.
  • Linked(external)- The style is in one .css file that is linked to multiple web pages. The link tag on the page is connected to the external file.
  • Properties: Properties are defined within selectors by defining a property and a value. They are separated with a colon and delineated with a semi-colon.
  • Padding: The padding is the spacing between the content and the border (edge of the element.).
  • Margin: The margin is the space around the element.
  • font-family: The font-family property sets the font of an HTML element’s text.
  • Selectors: Selectors are used in CSS to select the parts of the HTML that are being styled.
  • Class name selectors: You can also select HTML elements by their Class name. Unlike ID selectors, Class selectors select all elements with a matching class.
  • Element selectors: You are able to select HTML elements first by simply using the name of the element.
  • ID selectors: ID selectors are used to select only a single item on a page. Like the term (“identification”) indicates, ID selectors will ONLY select the first element with a matching ID.
  • Attribute selectors: HTML elements are also able to be selected by their attributes.
  • Child selectors:You can also use multiple selectors to get the exact elements you want, by using parental nesting. By using the “greater-than” symbol (>), you can select only the direct children of an element, going down only one level.
  • Universal selector:The universal selector (*) may be used to select all the elements in a particular range.
  • Pseudo class selectors:Pseudo Selectors can be used to narrow down a selection with certain rules.

Git

  • git status: Checks what branch we are currently on.
  • git checkout: -b branch-name: creates a new branch and switches to it.
  • git add: To add a file as it looks now to your next commit
  • git reset: Unstage a file while retaining the changes in working directory.
  • git diff: Diff of what is changed but not staged
  • git commit: Commit your staged content as a new commit snapshot
  • git branch: list your branches and will appear next to the currently active branch.
  • git merge: Merge the specified branch's history into the current one.
  • git merge name-of-branch: Will merge the specified branch into the branch you are currently on.
  • git branch -d name-of-branch-to-delete: Deletes the specified branch.
  • history: Will show you your past git commands.
  • git stash: Stashes any unstaged changes in your repository.
  • git stash pop: Gives you back the last staged changes you stashed
  • git clean -n: Test-Delete untracked files.
  • git clean -f: Delete untracked files.
  • git reset HEAD index.html: Unstage (undo adds)
  • git commit --amend -m "New Message": Update most recent commit (also update the commit message)
  • git merge branchname:True merge (fast forward).
  • git merge --ff-only branchname:Merge to master (only if fast forward)
  • git merge --no-ff branchname:Merge to master (force a new commit)

JavaScript

  • Variable: Is a container which has a name.
  • Loop: Can execute a block of code a number of times.
  • Numeric: Any numeric value, whether a whole number (an integer) or a number that includes a fractional part.
  • String: A group of text characters.
  • Boolean: A value which can only be either True or False
  • • When a new variable is created (or declared) its name must be preceded by the word var.
  • Operators: Is a type of command. They perform operations on variables and/or literals and produce a result.
  • Operators command: + = becomes equal to itself plus.
  • Operators command: - = becomes equal to itself minus.
  • Operators command: * = becomes equal to itself multiplied by.
  • Operators command: / = becomes equal to itself divided by.
  • Operators command: % = becomes equal to the amount which is left when it is divided by.
  • Properties: A variable (numeric, string or Boolean) associated with an object. Most properties can be changed by the user.
  • Methods: Functions associated with an object. Can be called by the user.
  • Events: Notification that a particular event has occurred. Can be used by the programmer to trigger responses.
  • Status:The contents of the status bar (at the bottom of the browser window.
  • Location:The location and URL of the document currently loaded into the window (as displayed in the location bar).Example: alert(window.location);
  • Top:The top-level window, of which all other windows are sub-windows. Example:var topWindow = window.top; alert(topWindow.length);
  • Length:The number of frames (if any) into which the current window is divided. Example: alert(window.length);
  • Parent:The parent window, if the current window is a sub-window in a frameset. Example: var parentWindow = window.parent; alert(parentWindow.length);
